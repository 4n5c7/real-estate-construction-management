# 不動産工事管理アプリ

不動産管理会社向けの工事案件管理ツール。受付〜見積〜発注〜完了〜請求までを一元管理する社内アプリです。

## 特徴

- **完全無償** — すべて OSS。月額費用ゼロ。
- **インストール簡単** — Python と Flask、SQLite だけで動作。
- **メンテしやすい** — シンプルな構成 (Flask + SQLite + openpyxl)
- **データは手元** — クラウド不要、ローカル/共有フォルダに保存
- **バックアップ簡単** — ZIP 1 ファイルで完結

## 必要環境

- Python 3.9 以上
- Web ブラウザ (Chrome / Edge / Firefox / Safari)
- ディスク空き容量: 100MB 以上

## 初回セットアップ

### 1. Python のインストール
[python.org](https://www.python.org/downloads/) から Python 3.9 以上をインストールしてください (「Add Python to PATH」にチェック)。

### 2. 依存ライブラリのインストール

```bash
cd /home/user/webapp
python -m pip install -r requirements.txt
```

(または起動スクリプト `start.bat` / `start.sh` が自動でインストールします)

### 3. 起動

| OS | コマンド |
| --- | --- |
| Windows | `start.bat` をダブルクリック |
| Mac / Linux | `./start.sh` を実行 |
| 直接実行 | `python run.py` |

ブラウザで `http://127.0.0.1:8000/` が自動で開きます。

## 主な機能

| 画面 | 機能 |
| --- | --- |
| ダッシュボード | KPI (進行中・完了・保留・緊急・期限超過)、状態別件数、操作履歴 |
| 工事一覧 | 検索・絞り込み・ソート、複数条件対応 |
| 工事登録/編集 | タブ式 (基本/スケジュール/見積/請求/添付/履歴/備考) |
| 添付管理 | 種別ごとに登録 (見積/請求/写真/承認/その他)、複数アップロード |
| マスタ管理 | 物件/業者/担当者/工事区分、論理削除 (無効化) |
| インポート | CSV/Excel 取り込み、新規マスタ自動候補化 |
| 出力 | 工事一覧表/オーナー報告/経理確認/業者別集計/CSV |
| バックアップ | ZIP 作成・復元、世代自動管理、整合性チェック |
| 設定 | 保存先パス、採番ルール、文字コード、添付サイズ上限 |

## データの保存場所

データはすべて `data/` フォルダに保存されます。**このフォルダを丸ごとコピーするだけで全データのバックアップ**になります。

### 共有フォルダ運用

複数 PC から共有フォルダのデータを使いたい場合：

1. 共有フォルダ (例: `\\server\share\工事管理`) を作成
2. アプリ起動後、設定画面で「データ保存先」を共有フォルダのパスに変更
3. 各 PC で同じパスを設定

> **注意**: SQLite は同時書き込みに弱いため、初期版では「同時編集しない」運用 (最終保存優先) を推奨します。

## バックアップ・復元

### 自動バックアップ
バックアップ画面の「今すぐバックアップ作成」ボタンで ZIP を作成できます。`data/backups/` 配下に保存され、保持世代数を超えたものは自動削除されます (既定: 30 世代)。

### 復元手順
1. バックアップ画面でファイル一覧から「復元」ボタンをクリック
2. 現状データは `before_restore_YYYYMMDD_HHMM.zip` として自動退避
3. 指定 ZIP の内容で上書き

## 工事 ID 採番

形式: `KOJI-YYYY-NNNN` (例: `KOJI-2026-0001`)

- 年は西暦4桁、連番は4桁ゼロ埋め
- 採番リセットは設定画面で「暦年」「年度 (4月開始)」を切替可能
- 採番テーブルでトランザクション制御 (重複なし)

## トラブルシュート

| 症状 | 対処 |
| --- | --- |
| ポート 8000 が使用中 | `python run.py --port 9000` のように別ポート指定 |
| ライブラリ読み込みエラー | `python -m pip install -r requirements.txt` を再実行 |
| 画面が真っ白 | `data/logs/error.log` を確認、ブラウザの開発者ツールでエラー確認 |
| データが消えた? | `data/backups/` の最新 ZIP から復元 |
| 文字化け (CSV) | 設定画面で「既定文字コード」を Shift_JIS に変更 |
